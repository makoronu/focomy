# 不動産テーマ企画書

**しろくま建物管理 物件サイト**

---

## 1. エグゼクティブサマリー

| 項目 | 内容 |
|------|------|
| プロジェクト名 | しろくま不動産テーマ |
| 目的 | 問い合わせ数の最大化 |
| KPI | CVR 5%以上（業界平均2.2%の2倍） |
| プラットフォーム | focomy CMS |
| データソース | REA（不動産管理システム）からAPI同期 |

---

## 2. 課題と解決策

### 現状の課題

| 課題 | 影響 |
|------|------|
| 物件情報がREAにしかない | 顧客が物件を閲覧できない |
| 問い合わせ導線がない | 機会損失 |
| スマホ対応が不十分 | 離脱率増加 |

### 解決策

| 解決策 | 効果 |
|---------|------|
| REA → focomy 自動同期 | 物件情報を公開サイトに表示 |
| CVR最大化設計 | 問い合わせ数増加 |
| モバイルファースト | スマホユーザー取り込み |

## 3. ターゲットユーザー

### ペルソナ

| 属性 | 詳細 |
|------|------|
| 年齢 | 30-50代 |
| 状況 | 住宅購入検討中 |
| デバイス | スマホ 70%、PC 30% |
| 行動 | 複数社比較、すぐに連絡したい |
| 不安 | 「この会社は信頼できるか？」 |

### ユーザージャーニー

```
検索「○○市 中古戸建」
    ↓
物件ページに到着（3秒で判断）
    ↓
写真を見る → 価格を確認
    ↓
「もっと知りたい」→ 問い合わせ
```

**勝負は最初の3秒**

## 4. 設計原則

### 根拠データ

| 施策 | 効果 | 出典 |
|------|------|------|
| CTAを1つに絞る | CVR 13.5% | Unbounce |
| ナビゲーション削除 | CVR 2倍 | VWO |
| フォーム5項目以下 | 離脱率半減 | HubSpot |
| パーソナライズCTA | CVR 202%向上 | HubSpot |
| 電話ボタン設置 | 不動産は38%が電話経由 | Promodo |
| 良いUX | CVR 400%向上 | Volts Consulting |

### 5つの原則

| # | 原則 | 理由 |
|---|------|------|
| 1 | **3秒ルール** | 3秒で「電話したい」と思わせる |
| 2 | **選択肢は最小** | 迷わせない（Apple方式） |
| 3 | **電話ファースト** | 不動産は電話が最強 |
| 4 | **入力は2項目** | 名前と電話番号だけ |
| 5 | **余白で信頼感** | シンプル＝プロ |

## 9. 検索機能（2層構造）

### 設計思想

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   🔍 「○○小学校区で庭付き、南向き」               │  ← AI検索（自然言語）
│                                                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│   基本フィルター（5つだけ）                         │  ← 従来型UI
│   [エリア][価格][種別][間取り][築年数]              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 基本フィルター（UI）

| # | フィルター | UI | 選択肢 |
|---|-----------|-----|--------|
| 1 | エリア | ドロップダウン | 市区町村リスト |
| 2 | 価格 | レンジスライダー | 500万〜5000万 |
| 3 | 種別 | チップ（複数選択可） | 中古戸建/土地/マンション |
| 4 | 間取り | チップ（複数選択可） | 2LDK以下/3LDK/4LDK以上 |
| 5 | 築年数 | ラジオボタン | 指定なし/10年/20年/30年 |

### AI検索（自然言語）

細かい条件はAI検索で対応:

| カテゴリ | 条件例 |
|---------|--------|
| **学校** | ○○小学校区、○○中学校区 |
| **交通** | ○○駅徒歩10分以内、バス停近く |
| **環境** | 南向き、日当たり良好、閑静 |
| **設備** | 駐車場2台、庭付き、ガレージ |
| **その他** | リフォーム済み、即入居可、ペット可 |

### AI検索の実装フェーズ

| Phase | 内容 | 難易度 |
|-------|------|--------|
| 1 | キーワードマッチング（備考欄LIKE検索） | ★☆☆ |
| 2 | 学校区フィールド追加（手動入力） | ★☆☆ |
| 3 | 学校区自動判定（住所→学校区マスタ照合） | ★★☆ |
| 4 | LLM自然言語検索（表記ゆれ対応、意図理解） | ★★★ |

### 検索UXの工夫

| 工夫 | 内容 |
|------|------|
| 件数リアルタイム表示 | フィルター変更で即「○件」更新 |
| 0件フィルター非表示 | デッドエンド回避 |
| 選択中フィルター可視化 | タグ表示、個別解除可能 |
| PC: 即時更新 | AJAX でフィルター変更時自動検索 |
| スマホ: 一括適用 | モーダル → 「○件を表示」ボタン |

### パフォーマンス要件

| 指標 | 目標 |
|------|------|
| 初回表示 | 1秒以内 |
| フィルター適用 | 500ms以内 |
| 件数更新 | 200ms以内 |

```
           │
           ▼
┌─────────────────────┐
│        REA          │

└─────────────────────┘
```

### 同期フロー

1. 管理画面で「今すぐ同期」クリック
2. REA APIから公開物件を取得
3. `rea_property_id`で差分検出
4. focomy `property`エンティティに保存
5. 結果表示（○件同期完了）

### property.yaml（案）

```yaml
name: property
label: 物件
icon: home
slug_field: property_name

fields:
  # 基本情報
  - name: property_name
    type: text
    label: 物件名
    required: true

  - name: rea_property_id
    type: number
    label: REA物件ID
    indexed: true

  - name: price
    type: number
    label: 価格（万円）

  - name: property_type
    type: select
    label: 物件種別
    options:
      - label: 中古戸建
        value: used_house
      - label: 土地
        value: land
      - label: 中古マンション
        value: used_apartment

  # 所在地
  - name: prefecture
    type: text
    label: 都道府県

  - name: city
    type: text
    label: 市区町村

  - name: address
    type: text
    label: 番地

  # 物件スペック
  - name: land_area
    type: number
    label: 土地面積（㎡）

  - name: building_area
    type: number
    label: 建物面積（㎡）

  - name: layout
    type: text
    label: 間取り

  - name: built_year
    type: number
    label: 築年

  - name: structure
    type: text
    label: 構造

  # 交通
  - name: access
    type: textarea
    label: 交通アクセス

  # 画像
  - name: images
    type: media
    label: 物件画像
    multiple: true

  # 地図
  - name: latitude
    type: number
    label: 緯度

    type: number

  # SEO
```

## 13. 公開終了ページ（SEO対策）

### 方針

物件が非公開になった場合、URLを維持して「公開終了」ページを表示する。

**理由**:
- 404より SEO に良い（バックリンクの価値維持）
- ユーザーを逃さない（類似物件へ誘導）
- 問い合わせ機会を残す

### 消す情報（プライバシー保護）

| 項目 | 対応 |
|------|------|
| 画像 | 非表示（プレースホルダー表示） |
| 詳細住所 | 市区町村まで表示、番地以下は消す |
| 経度・緯度 | 消す（地図非表示） |
| 価格 | 消す |
| 間取り詳細 | 消す |

### 残す情報（SEO・回遊用）

| 項目 | 理由 |
|------|------|
| エリア（市区町村） | 検索キーワードとして維持 |
| 物件種別 | 類似物件の検索に使う |
| 類似物件リスト | 離脱防止 |
| CTA（「似た条件で探す」「お問い合わせ」） | 問い合わせ機会維持 |
