# 振り返り（大プロンプト）

**作業完了後に実行。問題発生時は必須。**

---

## フロー

```
1_collect（事実収集）
├── timeline.md    ← 何が起きたか時系列
├── deviation.md   ← 逸脱ポイント特定
└── feedback.md    ← ユーザー指摘まとめ

2_analyze（原因分析）
├── prompt.md      ← プロンプト自体の問題
├── process.md     ← 運用の問題
├── root_cause.md  ← 根本原因
└── coe.md         ← 障害時はCOE記録

3_improve（改善）
├── proposal.md    ← 改善提案
├── update.md      ← プロンプト修正実施
└── report.md      ← 振り返り報告
```

---

## 実行条件

- **デプロイ完了後（必須）**
- 大きな問題が発生した後
- ユーザーから「振り返れ」と指示された時
- 1週間に1回（定期）
