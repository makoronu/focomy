# エラーログ（大プロンプト）

**全てのエラー報告と対応を記録。振り返りの基礎データ。**

---

## 必須ルール

1. **エラー発見時**: 即座にログ追記
2. **対応完了時**: ステータス更新
3. **デプロイ前**: 未対応エラーがないか確認

---

## ログ形式

```markdown
## [ERR-XXX] エラータイトル

| 項目 | 内容 |
|------|------|
| 報告日 | YYYY-MM-DD |
| 報告者 | GitHub Issue #N / ユーザー / 自己発見 |
| バージョン | vX.X.X |
| ステータス | 未対応 / 対応中 / 完了 / 保留 |
| 対応バージョン | vX.X.X |

### 症状
エラーメッセージ、再現手順

### 原因
根本原因

### 対応
修正内容、コミットハッシュ
```

---

## ログファイル

→ `docs/ERROR_LOG.md`

## 実行タイミング

- GitHub Issue コメント確認時
- ユーザー報告受領時
- 自己発見時
- デバッグ完了時（ステータス更新）
